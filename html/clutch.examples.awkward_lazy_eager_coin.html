clutch.examples.awkward_lazy_eager_coin

A minor variation of lazy_eager_coin.v where the coins take an additional function `f : 1 → 1` as argument, thus making it a variant of the Pitts & Stark "awkward example". It demonstrates the combination of local state encapsulation, asynchronous couplings via tapes, and higher-order functions. The proofs are the almost the same as in lazy_eager_coin.v, the only difference being the use of sequential composition with `f()` in the body of the closures.

From clutch Require Export clutch lib.flip.
Set Default Proof Using "Type*".

The lazy/eager coins, without tapes
Definition lazy : expr :=
  let: "s" := ref NONEV in
  λ: "f",
    "f" #() ;;
    match: !"s" with
    | NONE => let: "b" := flip in
              "s" <- SOME "b" ;;
              "b"
    | SOME "b" => "b"
    end.

Definition eager : expr :=
  let: "b" := flip in
  λ: "f", "f" #() ;; "b".

An intetermedaite version of lazy that uses a tape to allow presampling bits during the proof
Definition lazy_with_tape : expr :=
  let: "α" := allocB in
  let: "s" := ref NONEV in
  λ: "f",
    "f" #() ;;
    match: !"s" with
    | NONE => let: "b" := flipL "α" in
              "s" <- SOME "b" ;;
              "b"
    | SOME "b" => "b"
    end.
